> 数据来源：`h5/scripts/eval-balance.mts`（引擎仿真 + 公式采样）。仿真双方同 AI，仅反映数值走向，不完全等同真人体验。

## 关键数据

### 命中样本（75+(INT-AGI)×0.5）
- 李世民(INT88) 攻步兵(AGI35)：**99%**；长孙无忌(INT92) 攻步兵：**99%**（谋士物理命中倒挂）
- 敌步兵(INT25) 攻李世民(AGI78)：**49%**；攻长孙无忌(AGI65)：55%——敌方过半攻击落空，战斗无压力

### 伤害样本（Lv1，普通，平原）
- 李世民(STR82) → 隋军步兵(CMD40/HP45)：26/击，2 击击杀（合理）
- 隋军步兵(STR45) → 李世民(CMD95)：**1/击**（保底），李世民可抗 120 击（免伤）
- 隋军步兵 → 长孙无忌(CMD60/HP80)：1/击，谋士也不存在生存压力

### 难度仿真（5 种子，AI 自动）
- 第 1 关：story 2.0 回合 / normal 3.0 / hard 3.0
- 第 7 关：story 2.0 / normal 2.0 / hard 7.2——normal 与 story 无区分度

### 关卡曲线（normal，推荐等级）：1 关 3.0 → 5 关 11.0 → 7 关 2.0 回合，全程无败绩

## 调整建议（按性价比排序，未实施，待拍板）

1. **敌军数值强化**（数据层，风险最小）：敌军模板 STR 整体 +15~25（或按关卡递增：关 n 敌军全属性 +(n-1)×3），让敌攻能穿透我方 CMD；Boss 再加成。工作量小，直接解决"敌人零输出"。
2. **伤害公式防御半减**：`STR×克制 - CMD×0.5`，高防单位从免伤变为减伤一半；同时命中公式 INT→STR/AGI 混合（如 `75+(STR×0.3+AGI×0.2-守AGI)×0.5`）修正谋士倒挂。**改公式影响全局手感，需重新仿真调优**。
3. **难度档拉开**：normal 不动，story 再降敌数值（×0.6/×0.65），easy 并入 story 与 normal 之间，hard 加敌增援/AI 集火增强——让 4 档都有感知。
4. **关卡时长**：与 1/2 联动（敌人更能打自然拖长回合）；后续可加文档原有的增援/多阶段事件（turnEvents 字段已预留）。

> 建议先做 1（纯数据、可快速验证），试玩后再决定是否动 2 的公式。

## 调整实施记录（2026-07-17，已上线）

实施方案（组合拳）：**伤害公式防御半减**（`COMBAT_FORMULA.defenseCoef=0.5`，普攻与落石/乱射同口径）+ **敌军按关卡递增强化**（`battle.ts makeEnemyUnit`：第 n 关敌军 STR+4(n-1)、CMD+3(n-1)、AGI+2(n-1)、HP+10(n-1)，第 1 关教学基准不加成）。

调整后关键数据：
- 隋军步兵 → 李世民：**7/击**（原 1）；→ 长孙无忌：**15/击**（原 1，5 击可杀，谋士需保护成立）
- 第 7 关难度区分：story 2.0 / normal 2.0 / **hard 5.0 回合**（hard 开始拉开）
- 8 关仿真 24/24 仍全胜、AI 场均我方阵亡 0——AI 用推荐等级满编集火，参考上限；真人非最优操作下压力会显著高于仿真
- 命中公式倒挂（谋士 INT 驱动物理命中）**暂未改**：谋士物理伤害极低，倒挂仅为观感问题，改动会牵连全局命中手感，留待下一轮
- 依据 PRD §8「失败惩罚低、不设地狱级、轻松看故事」的定位，当前难度**刻意不再加码**；若真人试玩仍觉过易，下一步开关：敌军递进系数（STR+4→+6/HP+10→+14）或启用困难档敌增援
