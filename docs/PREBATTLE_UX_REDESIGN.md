# 战前准备流程 UX 重设计 v2.0

> 将原来"选关→直接编组"的一步式改为**三级页面**流程：
> 选关 → 选人 → 配装。
> 审查通过后进入开发。

---

## 1. 当前问题

| 问题 | 说明 |
|------|------|
| 角色自动填充 | 当前从 `availableCharacters` 自动取前8人，玩家无法选择谁上场 |
| 信息过载 | 编组界面同时展示角色列表+装备+被动+羁绊，视觉负担重 |
| 字体太小 | 角色名14×_scale，属性11×_scale，手机上看不清 |
| 缺乏勾选交互 | 玩家无法直观地勾选/取消勾选上场角色 |

---

## 2. 新流程设计

```
关卡选择 (LevelSelect)                    ← 已有，不修改
    │
    │ 点击解锁关卡
    ▼
选人界面 (HeroSelect)                     ← 新增
    │ 展示所有可用角色 + 勾选上场
    │ 必出角色锁定不可取消
    │ 确认后进入装备
    │
    ▼
装备调整 (EquipSetup)                    ← 重写
    │ 左: 已选角色列表 (点击切换)
    │ 右: 装备卡 + 五维 + 被动 + 羁绊
    │ 底部: 开始战斗
    │
    ▼
战斗 (Battle)
```

### 3.2 角色卡片

每行一个角色，高度 80×_scale：

```
┌──────────────────────────────────────┬────┐
│ 兵种色条 │ 李世民        Lv.3 骑兵  │ ✅ │
```

| 元素 | 字号 | 说明 |

| 兵种色条 | 4×_scale 宽 | 左边缘竖条，颜色=兵种色 |
| 角色名 | **22×_scale Bold** | 白色 |

| 勾选框 | **30×_scale** | ✅ 已选 / ☐ 未选 |

### 3.3 勾选逻辑

| 规则 | 说明 |
|------|------|
| 必出角色 | `LevelData.requiredCharacters` 中定义，自动勾选且禁用取消 |
| 可选角色 | `LevelData.availableCharacters` 中定义，默认前N个勾选到满8人 |
| 上限 | 最多 **8 人**，达到上限后未勾选的变为灰色不可选 |
| 下限 | 至少保留必出角色（通常至少1人） |
| 勾选变化时 | 右侧羁绊面板实时更新 |

### 3.4 羁绊预览

勾选变化时，下方羁绊面板**实时更新**：

| 状态 | 显示 |
|------|------|
| 羁绊条件全部满足 | 金色高亮 "👑 帝后同心 已激活：全属性+10%" |
| 羁绊部分满足 | 灰色 "⚔ 瓦岗三杰 还需程咬金 全队攻击+12%" |
| 无羁绊 | 不显示 |

### 3.5 空状态

- 所有可选角色已满8人时，未选角色显示"阵容已满"不可勾选
- 可用角色少于必出角色时，显示"可用角色不足"

### 3.6 按钮

| 按钮 | 位置 | 行为 |
|------|------|------|
| ← 返回选关 | 左下 | 回到关卡选择（不保存） |
| 确认阵容 → | 右下 | 进入装备调整界面 |

## 6. 状态与异常处理

### 阵容不可确认
- 必出角色不足 → "缺少必出角色" 提示
- 已选角色超过上限 → 已达上限，需取消其他角色

### 装备不可用
- 某槽位无可装备物品 → 显示"无可用装备"灰色提示
- 角色职业限制 → 装备列表自动过滤（已有逻辑）

## 7. 技术实现方案

### 7.1 新增 GamePhase

```csharp
// 在 GamePage 枚举中增加
public enum GamePage
{
    Splash, MainMenu, Settings, LevelSelect,
    HeroSelect,       // ← 新增：选人界面
    EquipSetup,       // ← 新增：装备调整（替代PreBattle）
    Story, Battle, Results
}
```

### 7.2 BattleTestController 内部状态

```csharp
private enum BattlePhase
{
    LevelSelect, HeroSelect, EquipSetup, Battle, Results
}
```

### 7.3 新增方法

| 方法 | 职责 |
|------|------|
| `DrawHeroSelectUI()` | 渲染选人界面 |
| `DrawEquipSetupUI()` | 渲染装备调整界面（替代 DrawPreBattleUI） |
| `DrawEquipCard()` | 渲染单个装备卡 |
| `DrawStatComparison()` | 渲染五维对比三列表 |
| `DrawBondPreview()` | 渲染羁绊实时预览（选人页） |
| 重写 `DrawEquipSlot()` | 改为装备卡样式 |
| 重写 `DrawEquipSelectionList()` | 增大行高+稀有度色 |

### 7.4 流程控制

```
LevelSelect → 点击关卡
  → 检查关前剧情（Story）
  → HeroSelect（选人）
     → 勾选/取消 → 实时更新羁绊预览
     → 确认阵容 → EquipSetup（装备调整）
        → 调整装备 → 开始战斗（Battle）
```

### 7.5 过渡保护

- HeroSelect ↔ LevelSelect 之间：直接切换，无过渡动画（当前已禁用过渡保护）
- HeroSelect → EquipSetup：保存已选阵容
- EquipSetup → Battle：初始化战斗
