# 贞观勇士 H5 重建设计文档 v0.1

> 目标：放弃 Unity 客户端体验差的包袱，用 H5 技术栈重建 MVP，尽快产出可对外试玩的版本。
> 本文档是 H5 版（`h5/` 目录）的唯一设计依据，开发严格按此执行。

## 1. 背景与目标

- Unity 版功能虽已接近完整（8 关/15 角色/30 装备），但 OnGUI 体验差、打包/分发链路重、试玩成本高。
- H5 版核心卖点：**点开链接即玩**，无需安装，适合外部试玩与快速迭代。
- 本阶段目标：**MVP 可试玩**（完整游戏闭环 + 8 关 + 15 角色），不是 Unity 版全功能平移。

### 1.1 成功标准（验收）
- 手机浏览器打开 URL，3 秒内进主菜单，竖屏可玩。
- 完整闭环：主菜单 → 选关 → 剧情 → 选人 → 配装 → 战斗 → 结算 → 下一关，8 关可通关。
- 战斗规则按 `02-combat.md` 实现（克制/地形/天气/计策/命中/暴击/反击），AI 可正常行动。
- 存档存 localStorage，刷新页面可继续。
- 构建产物为纯静态文件（`dist/`），可部署到任意静态托管。

## 2. 技术选型

| 层面 | 选型 | 理由 |
|---|---|---|
| 语言 | TypeScript 5 | 类型安全，战棋数值/状态复杂，纯 JS 易失控 |
| 构建 | Vite 5 | 秒级 HMR，产物为纯静态文件 |
| 战斗渲染 | **Canvas 2D**（自绘，无引擎） | 六边形网格+圆形棋子+简单动画，Canvas 足够；不引 Phaser/Cocos（包体、学习成本、AI 协作成本） |
| UI 界面 | **DOM + CSS**（原生，无框架） | 菜单/选人/配装/剧情/结算都是列表+按钮+文本，DOM 的文本排版、滚动、中文渲染、适配都最省事 |
| 状态/路由 | 单例 Game 类 + 页面状态机 | 沿用 `UX_DESIGN.md` 的 GamePhase 枚举，任何时刻只显示一个页面 |
| 存档 | localStorage（JSON） | 单机游戏无需后端 |
| 音频 | 暂缓（MVP 静音，留 AudioManager 接口） | 占位音频对试玩价值低，二期再接 |
| 依赖 | **零运行时依赖** | 只有 vite + typescript 两个 devDependency |

手机竖屏优先（设计基准 390×844，iPhone 12/13/14 档位），桌面浏览器兼容（居中 420px 宽栏 + 可鼠标操作）。

## 3. MVP 范围

### 3.1 包含（P0）
- 页面流：启动页 → 主菜单 → 选关 → 关前剧情 → 选人 → 配装 → 战斗 → 关后剧情 → 结算（下一关/重试/选关）
- 战斗引擎：六边形网格（flat-top，Cube 坐标）、A* 寻路（地形消耗+占据格）、移动/攻击/计策/待机、命中/伤害/暴击/反击公式、8 兵种克制表、9 种地形、5 种天气、10 种计策、胜负判定（全灭/击破主将/坚守回合/回合上限/主角阵亡）
- 敌方 AI：`02-combat.md` 行为树优先级版（撤退→计策→集火→逼近）
- 15 名角色（数值按 §4.1 表）+ 升级成长 + 30 件装备（按 Unity `EquipmentData.cs` 移植）+ 6 组羁绊（编组时实时提示，战斗中属性加成）
- 8 个关卡（地形/部署用 Unity `level_0X.json` 实装版，奖励/敌人数值按文档校准）
- 剧情对话：8 关关前/关后（文本从 `01-story.md`/`StoryData.cs` 移植，打字机效果，点击快进，无选项分支）
- 存档：自动存档（过关后）+ 继续游戏 + 新游戏；难度 4 档随时可调；角色等级/装备/资金跨关保留
- 新手引导：第 1 关内嵌 4 条操作提示（选中→移动→攻击→结束回合）

### 3.2 明确不做（二期再议）
- 单挑系统、商店/打造、转职、招募、激励视频广告
- 剧情选项分支与忠诚度（对话纯线性播放）
- 关卡内剧本事件（尉迟倒戈、增援、决堤等 turnEvents，8 关按静态部署打）
- BGM/SFX（留接口）、云存档、横屏、平板专项适配、微信小游戏
- 战斗中存档/悔棋（MVP 战斗为一次性会话，撤退=重打）

> 范围裁剪原则：先保证"能玩完 8 关且好玩"，一切不影响闭环的系统后置。

## 4. 数据方案（单一事实来源）

所有游戏数据写成 TS 模块（`h5/src/data/`），类型定义见 `h5/src/core/types.ts`：

| 数据 | 来源（移植自） | H5 文件 |
|---|---|---|
| 15 角色（五维/成长/技能/被动） | `03-character.md` 表 + Unity `CharacterDatabase.cs` | `data/characters.ts` |
| 30 装备 | Unity `Core/Combat/EquipmentData.cs` | `data/equipment.ts` |
| 10 计策 + 25 被动技能 | `02-combat.md` + Unity `SkillData.cs`/`PassiveSkillData.cs` | `data/skills.ts` |
| 6 组羁绊 | `03-character.md` + Unity `BondSystem.cs` | `data/bonds.ts` |
| 8 关卡（地形/敌军/部署） | Unity `Assets/Data/Levels/level_0X.json` 实装值 | `data/levels.ts` |
| 地形/天气/克制系数 | `02-combat.md` 表 | `data/rules.ts` |
| 8 关剧情对话 | Unity `Core/Story/StoryData.cs` + `01-story.md` | `data/story.ts` |

数值冲突时优先级：**文档（02/03/04）> Unity JSON > Unity C#**。已知差异（关5 天气、关6-8 尺寸等）以 Unity JSON 实装为准，文档的剧本事件一律不做（见 §3.2）。

## 5. 架构设计

```
h5/
├── index.html                  # 唯一页面，#app 容器
├── package.json / vite.config.ts / tsconfig.json
├── public/                     # 静态资源（favicon 等）
└── src/
    ├── main.ts                 # 入口：new Game().start()
    ├── core/                   # 纯逻辑层：零 DOM 依赖，可在 node 下跑测试
    │   ├── types.ts            # 全部类型定义（唯一事实来源）
    │   ├── hex.ts              # Cube 坐标、邻居、距离、范围、像素转换
    │   ├── pathfinding.ts      # A*（地形消耗 + occupiedCells）
    │   ├── rules.ts            # 命中/伤害/暴击/反击/经验 公式（纯函数）
    │   ├── battle.ts           # BattleState + 回合机 + 行动执行 + 胜负判定
    │   ├── ai.ts               # 敌方行为树
    │   ├── save.ts             # localStorage 读写 + 版本迁移
    │   └── sim.ts              # 无头仿真：自动打一场战斗（验证用）
    ├── data/                   # 数据表（见 §4）
    ├── ui/                     # DOM 页面层（每页一个类，render(root) / destroy()）
    │   ├── splash.ts / mainMenu.ts / levelSelect.ts
    │   ├── story.ts            # 对话播放（打字机）
    │   ├── heroSelect.ts / equipSetup.ts / results.ts / settings.ts
    │   └── game.ts             # 页面状态机（GamePhase 路由）
    ├── battle/                 # Canvas 战斗层
    │   ├── battleScene.ts      # 场景总控：渲染循环 + 输入分发 + 与 core/battle 对接
    │   ├── renderer.ts         # 网格/地形/单位/血条/范围高亮/路径 绘制
    │   ├── camera.ts           # 平移 + 捏合/滚轮缩放
    │   └── anim.ts             # 移动补间/攻击闪白/伤害飘字
    └── styles/main.css         # 全局样式 + 唐风色板
```

### 5.1 关键接口契约

```ts
// core/battle.ts —— UI 与引擎的唯一交互面
class Battle {
  constructor(level: LevelData, party: PartyMember[], difficulty: Difficulty)
  state: BattleState            // 只读快照：units/turn/phase/weather/terrain
  // 玩家意图（全部返回 ActionResult，含动画所需的事件序列）
  selectUnit(id: string): Selection        // 可达格/可击格
  moveAndAttack(unitId, path, targetId?): ActionResult
  useSkill(unitId, skillId, targetCell): ActionResult
  wait(unitId): ActionResult
  endTurn(): ActionResult       // 跑敌方+友方整回合，返回完整事件流
  isOver(): BattleResult | null // 胜负
}
// ActionResult = { events: BattleEvent[] }，事件流驱动动画，UI 不直接改状态
```

- 动画与逻辑解耦：`core` 同步算完一整个行动/回合，把"发生了什么"作为事件序列（move/attack/damage/die/skill/buff/turnBegin...）交给 `battleScene` 依次播放。
- 存档只存元进度（关卡进度/角色等级经验/装备/资金/难度），不存战斗中状态。

### 5.2 页面状态机（沿用 UX_DESIGN）
`Splash → MainMenu → LevelSelect → Story(pre) → HeroSelect → EquipSetup → Battle → Story(post) → Results → LevelSelect/下一关`；Settings 从 MainMenu 进出。

## 6. UI/UX 规范（H5 版）

- 色板沿用 PRD §9.4：朱红 `#B7261E` / 宣纸黄 `#F5EBD1` / 金 `#E6BF33` / 深褐底 `#1F140D` / 面板 `#332318` / 卡片 `#473320`；兵种色与范围色用 `06-ui-ux.md` 色板。
- 字号（CSS px，DOM 自动 DPI 适配，无需 Unity 那套补偿）：页面标题 28 / 卡片标题 20 / 正文 16 / 辅助 13。
- 触控目标 ≥ 44px；底部主按钮宽 100% 高 52px；列表卡片全宽。
- 战斗画面：顶部回合栏（回合数/天气/菜单）32px；中央 Canvas 战场（单指拖拽平移、双指捏合缩放 0.6–1.6×、点击选中）；选中己方单位后底部弹行动面板（攻击/计策/待机/取消 + 结束回合常驻）；点敌人显示攻击预览（命中率/伤害区间/暴击率 → 确认/取消）。
- 所有页面安全区适配 `env(safe-area-inset-*)`。

## 7. 里程碑

| 里程碑 | 内容 | 验收 |
|---|---|---|
| M0 设计 | 本文档 + 脚手架 | `npm run dev` 起空白页 |
| M1 引擎 | core/ 全部 + 数据表 + 无头仿真 | `npm run sim` 自动打完第 1 关且有胜负 |
| M2 战斗画面 | Canvas 渲染 + 触控交互 | dev 服务器里手动打赢第 1 关 |
| M3 页面流 | 全部 UI 页面 + 剧情 + 存档 | 新游戏→通关第 1-2 关→刷新后 continue |
| M4 可试玩 MVP | 8 关数据全接入 + 难度 + 打磨 | `npm run build` 通过，dist 可部署，完整通关链路 |
| M5 部署 | 静态托管上线 | 手机浏览器可访问（部署平台待用户确认） |

## 8. 部署方案（待用户确认）

产物是纯静态 `dist/`。候选：① 腾讯云 COS/阿里云 OSS 静态网站 + CDN（国内访问快，需用户提供账号）；② Vercel/Netlify（零配置但国内访问不稳）；③ 临时方案：局域网/内网穿透试玩。域名/备案另行决策。MVP 构建完成后由用户拍板。

## 9. 风险与对策

| 风险 | 对策 |
|---|---|
| 战斗公式移植出偏差导致失衡 | core 层纯函数 + `sim.ts` 无头仿真自动对战，数值问题先暴露在无 UI 环境 |
| Canvas 触控手感差（点不准格子） | 格子命中判定放大 1.2×；选中后二次确认；相机缩放下限放宽 |
| 中文渲染/字体 | 系统字体栈（"PingFang SC","Microsoft YaHei"），不内嵌字体文件 |
| 8 关数值未实测 | M4 用 sim 批量自动通关验证难度曲线，再人工试玩调优 |
| 一次性交付过大 | 按 M1→M4 顺序集成，每个里程碑可独立验收 |
