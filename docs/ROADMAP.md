# 贞观勇士 开发路线图 v0.2（拆解版）

## 总原则：垂直切片先行

先打通一条"选关→编组→战斗→结算→下一关"的完整链路（只做1关、2个角色），
再横向扩充（更多关、更多角色、更多系统）。每期产出可运行的 APK。

**目标**：兵种相克 + 计策系统 + 升级 AI，战斗有策略性

## Sprint 8：MVP 缺口补齐与稳定性加固 ✅

**目标**：解决当前版本与 MVP 的实际差距，补齐关卡 JSON、剧情接入、友军 AI、音效启用、装备奖励、防御性编程等，使核心玩法循环完整可用。

| 优先级 | 模块 | 状态 | 问题描述 | 验收标准 |
|---|---|---|---|---|
| P0 | 关卡 JSON 导出 | ✅ | `Assets/Data/Levels/` 下只有 `level_01.json`，第2–8关依赖硬编码 | 8 关全部导出为 JSON，运行时 `LoadFromResources` 成功加载 |
| P0 | 剧情流程接入 | ✅ | `SelectLevel()` 直接进 HeroSelect，关前/关后剧情被跳过 | 选关后先播放 `story_{levelId}_pre`，胜利后播放 `story_{levelId}_post` |
| P0 | 友军 AI 兜底 | ✅ | `TurnManager` 定义了 `AllyTurn` 但 `CheckPhaseEnd` 无处理分支 | 含友军关卡不卡死；暂无 AI 时自动 Skip |
| P1 | 音效系统启用 | ✅ | `AudioManager` 播放逻辑全部注释，游戏静音 | 导入占位音频后，BGM/SFX 正常播放 |
| P1 | 单位名字标签 | ✅ | `UnitVisual.UpdateNameVisibility()` 直接隐藏名字 | 战场上能显示/查看单位名称 |
| P1 | 装备获取途径 | ✅ | 装备系统完整但只来自初始默认 | 通关获得金钱/战利品，或战前商店购买 |
| P2 | 防御性编程 | ✅ | `GetComponent`、`_hexView`、`_turnManager`、`Camera.main` 多处裸用 | 关键路径增加判空，异常时给出中文提示而非崩溃 |
| P2 | 协程安全 | ✅ | `MoveUnitAnimation` 运行期间单位阵亡可能修改字典 | 移动/死亡协程互斥，避免迭代异常 |
| P2 | 存档兼容性 | ✅ | `JsonUtility` 字段变更时老存档可能静默丢失 | 版本号校验 + 字段缺失兜底 |

**验证方法**：完整通关 8 关，期间 Android 返回键、暂停存档、单位重叠、剧情对话、音效均正常。

### 已知风险清单

| 风险 | 影响 | 当前缓解 |
|---|---|---|
| 快速连点导致页面切换穿透 | 可能同时打开两个页面 | `IsTransitioning` 标志，但需补充输入屏蔽时间 |
| 初始布置重叠 | `ResolveUnitOverlaps` 是事后修复 | 已增加 `CheckOverlapAt` 日志，需测试极端关卡 |
| 内存泄漏 | `RebuildFromLevelData` 销毁旧 GameObject | 材质/纹理未显式清理，长期切换关卡需关注 |
| 老存档不兼容 | 新增字段反序列化失败 | `SAVE_VERSION` 升级到 2，`LoadFromPath` 增加版本迁移与字段缺失兜底 |
