# 贞观勇士 开发路线图 v0.2（拆解版）

## 总原则：垂直切片先行

先打通一条"选关→编组→战斗→结算→下一关"的完整链路（只做1关、2个角色），
再横向扩充（更多关、更多角色、更多系统）。每期产出可运行的 APK。

---

## Sprint 1：核心战斗 Demo ✅（已完成）

**目标**：跑通一回合攻防，验证战棋手感

- [x] Unity 2022.3 LTS 项目创建，URP 2D 配置
- [x] 六边形网格（Cube坐标系，Pointy-top）
- [x] A* 寻路（地形消耗支持）
- [x] 回合控制器（己方→敌方→循环）
- [x] 战斗单位状态机
- [x] 基础伤害计算（命中/暴击/地形修正）
- [x] 点击交互（选人→移动→攻击）
- [x] 简易 AI（寻路 + 攻击最近目标）
- [x] 圆形 Sprite 单位 + HP 血条 + 选中高亮
- [x] 回合 UI 文字

**产出**：APK，能看到地图和单位，可操作移动攻击

---

## Sprint 2：战斗系统深度

**目标**：兵种相克 + 计策系统 + 升级 AI，战斗有策略性

| 模块 | 内容 |
|---|---|
| **兵种系统** | 定义 6 兵种（步/重步/骑/弓/器械/谋士），兵种相克表，兵种标签显示 |
| **计策系统** | 计策数据模型 → 火攻(AOE) → 医疗(单体) → 鼓舞(Buff) → 落石(地形)，MP消耗 |
| **AI 升级** | AI 会使用计策、谋士保持后排、残血撤退 |
| **武将单挑** | 相邻触发→武力对比→斩杀 |

**验证方法**：同一张地图，重步克制骑兵打出的伤害不一样，谋士能放火攻烧一片

---

## Sprint 3：角色成长 + 装备 ✅（已完成）

**目标**：角色有差异、可培养

| 模块 | 状态 | 内容 |
|---|---|---|
| **装备系统** | ✅ | 30件装备数据结构(武器/防具/饰品) → 装备影响五维/HP/MP/移动/射程 → 战前编组UI(OnGUI)可自由调整 |
| **经验升级** | ✅ | 杀敌得经验 → 升级涨五维(成长率) → 技能习得框架(角色ID+等级对照表) |
| **兵种特性** | ✅ | 骑兵突击(平原+20%伤害+1射程) 弓兵远射(+1射程) 器械破阵(普攻溅射50%) 谋士医疗强化(+30%) |

**验证方法**：角色升1级后攻击力上升，装上武器后伤害变化

---

## Sprint 4：关卡 + 多角色阵容 ✅（已完成）

**目标**：可以选关，有6个可用角色

| 模块 | 状态 | 内容 |
|---|---|---|
| **关卡数据** | ✅ | 8个关卡(晋阳→霍邑→长安→浅水原→柏壁→洛阳→虎牢关→玄武门)完整定义，支持JSON配置+硬编码双加载 |
| **关卡选择** | ✅ | 大地图关卡选择界面(OnGUI)，显示关卡名/条件/敌人数/天气，锁定/解锁机制 |
| **胜利条件** | ✅ | VictoryChecker完整实装：全灭敌军、击破主将、坚守回合、到达点、主角存活 + 5种失败条件 |
| **关卡切换** | ✅ | 通关→结算界面(经验统计+存活统计)→下一关/重试/返回选关，逐关解锁 |
| **角色阵容** | ✅ | 每关availableCharacters+requiredCharacters控制出场阵容 |

**验证方法**：打完第1关（晋阳起兵）进入第2关（霍邑）

---

## Sprint 5：剧情对话 + 主菜单 + 存档 ✅（已完成）

**目标**：完整的游戏闭环

| 模块 | 状态 | 内容 |
|---|---|---|
| **主菜单** | ✅ | 标题画面、新游戏/继续游戏/设置三按钮、版本号显示 |
| **对话系统** | ✅ | StoryData对话数据模型 + DialogueUI渲染(说话人/头像占位/对白/自动或选择推进)、8场关前/关后剧情 |
| **存档系统** | ✅ | SaveData+SaveManager JSON序列化、自动存档(通关后)、新游戏清档、从存档恢复角色等级/装备/技能 |
| **关卡选择** | ✅ | 已有（来自Sprint 4），与存档进度联动 |
| **编组界面** | ✅ | 已有（来自Sprint 3）+ 从存档恢复角色状态 |
| **战斗结算** | ✅ | 已有（来自Sprint 4）+ 关后剧情 + 自动存档 |

**验证方法**：新游戏→看剧情→打关→存档→退出→继续→从存档续玩
