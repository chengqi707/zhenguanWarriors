# 贞观勇士 开发路线图 v0.2（拆解版）

## 总原则：垂直切片先行

先打通一条"选关→编组→战斗→结算→下一关"的完整链路（只做1关、2个角色），
再横向扩充（更多关、更多角色、更多系统）。每期产出可运行的 APK。

---

## Sprint 1：核心战斗 Demo ✅（已完成）

**目标**：跑通一回合攻防，验证战棋手感

- [x] Unity 2022.3 LTS 项目创建，URP 2D 配置
- [x] 六边形网格（Cube坐标系，Pointy-top）
- [x] A* 寻路（地形消耗支持）
- [x] 回合控制器（己方→敌方→循环）
- [x] 战斗单位状态机
- [x] 基础伤害计算（命中/暴击/地形修正）
- [x] 点击交互（选人→移动→攻击）
- [x] 简易 AI（寻路 + 攻击最近目标）
- [x] 圆形 Sprite 单位 + HP 血条 + 选中高亮
- [x] 回合 UI 文字

**产出**：APK，能看到地图和单位，可操作移动攻击

---

## Sprint 2：战斗系统深度

**目标**：兵种相克 + 计策系统 + 升级 AI，战斗有策略性

| 模块 | 内容 |
|---|---|
| **兵种系统** | 定义 6 兵种（步/重步/骑/弓/器械/谋士），兵种相克表，兵种标签显示 |
| **计策系统** | 计策数据模型 → 火攻(AOE) → 医疗(单体) → 鼓舞(Buff) → 落石(地形)，MP消耗 |
| **AI 升级** | AI 会使用计策、谋士保持后排、残血撤退 |
| **武将单挑** | 相邻触发→武力对比→斩杀 |

**验证方法**：同一张地图，重步克制骑兵打出的伤害不一样，谋士能放火攻烧一片

---

## Sprint 3：角色成长 + 装备

**目标**：角色有差异、可培养

| 模块 | 内容 |
|---|---|
| **装备系统** | 武器/防具/饰品 数据结构 → 装备影响属性 → 战前编组装备 |
| **经验升级** | 杀敌得经验 → 升级涨五维 → 技能习得框架 |
| **兵种特性** | 骑兵突击距离+1、弓兵射程+1、器械AOE、谋士远程 |

**验证方法**：角色升1级后攻击力上升，装上武器后伤害变化

---

## Sprint 4：关卡 + 多角色阵容

**目标**：可以选关，有6个可用角色

| 模块 | 内容 |
|---|---|
| **关卡数据** | JSON 配置（地图/敌兵/胜利条件/剧情触发） |
| **6 角色实装** | 李世民（骑）、李靖（骑）、尉迟敬德（重步）、秦琼（骑）、长孙无忌（谋士）、平阳公主（骑） |
| **胜利条件** | 全灭敌军 / 击破主将 / 守住X回合 |
| **关卡切换** | 通关→结算→下一关 |

**验证方法**：打完第1关（晋阳起兵）进入第2关（霍邑）

---

## Sprint 5：剧情对话 + 主菜单 + 存档

**目标**：完整的游戏闭环

| 模块 | 内容 |
|---|---|
| **主菜单** | 新游戏 / 继续 / 设置 |
| **对话系统** | 立绘 + 文本 + 选项→影响关卡 |
| **存档系统** | 自动存档 + 5 手动槽位 |
| **大地图** | 关卡选择界面，锁定的关不可选 |
| **编组界面** | 12选8，调整装备 |
| **战斗结算** | 经验统计、掉落物展示 |

**验证方法**：新游戏→看剧情→打关→存档→退出→继续→从存档续玩
