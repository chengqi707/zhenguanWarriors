# 贞观勇士 PRD v0.1（MVP）

## 1. 立项背景
向《曹操传》（光荣 1998）致敬的国产历史战棋 RPG 手游。主人公李世民，时代背景为隋末唐初。MVP 验证战棋核心玩法和历史题材在国内移动端的可行性。

## 2. 目标用户
- 25-45 岁，怀旧 SLG 玩家，熟悉曹操传/三国志系列
- 历史爱好者，偏好单机/弱联网体验
- 通勤碎片时间玩家，可接受 20-30 分钟一关

## 3. 核心玩法
- 六边形格子战棋 + 回合制 RPG
- 兵种相克（步/骑/弓/器械/谋士）、计策（火攻/落石/水攻/乱射/鼓舞/医疗）
- 武将单挑、宝物装备、经验升级、技能习得
- 剧情对话推进，关卡间隙触发事件
- 角色类型多样化：君主、武将、谋士、女性角色，各有定位

## 4. MVP 范围（v0.1）
- 8 个剧情关卡：起兵晋阳→霍邑→长安→浅水原→柏壁→洛阳→虎牢关→玄武门前夜
- **15 名可上阵角色**（含 8 武将 + 4 谋士 + 1 君主 + 2 女性，上阵选 8）
- 兵种 5 种、计策 8 种、装备 30 件、技能 25 个
- 主菜单/大地图/战斗/结算/存档完整链路

## 4.2 新兵种「谋士」设计
- HP：低（70-90，仅武将一半）
- 移动力：中（5格）
- 物理攻击：极低（10-25）
- 物理防御：低（15-30）
- 计策伤害：高（基于智力60-95）
- 计策抗性：高
- 普通攻击范围：2格（持卷/扇/笏远距离）
- 兵种克制：克制"器械"，被"骑兵"克制
- 必须保护：阵亡后影响士气，全队攻击-10%
- 解锁的计策：火攻、水攻、乱射、混乱、鼓舞、医疗、复活（皇后限定）、洞察

## 5. 优先级（P0-P4）
- P0：战棋引擎、战斗、4 关、6 角色（李世民/李靖/尉迟/秦琼/长孙无忌/平阳公主，含谋士女性兵种验证）、存档
- P1：剧情对话、装备、技能、其余 4 关、其余 9 角色（含房玄龄/杜如晦/长孙皇后）
- P2：单挑、计策完整、宝物商店、武将羁绊（帝后/夫妻/三杰）
- P3：BGM/SFX、深色UI、设置
- P4：分支结局、成就、云存档

## 7. 战斗系统细则
- 命中率 = 武将命中 - 目标回避 + 地形修正
- 伤害 = (攻×兵种克制) - 防 ± 随机5%
- 计策伤害基于智力，受目标抗性
- 单挑触发：双方武将相邻且武力差≤15，胜方斩杀败方部队
- 地形：平原/林/山/水/城/桥，影响移动力和命中

## 8. 数值平衡原则
- 主角等级 30 通关；普通武将 25 级
- 单关 15-30 分钟
- 失败惩罚低：可重打不扣资源
- 友军全灭/主角阵亡=失败；指定敌将击破=胜利
- **全局难度 4 档**（极简/简单/普通/困难），任意时刻可调，覆盖从纯剧情党到资深玩家
- **不设地狱级**——产品定位是"轻松看完故事+体验战棋乐趣"，不做需要读档的高墙关卡

## 8.1 存档总则（产品级硬要求）
- **任何关卡内任何时候**可手动存档退出，5 个手动槽位
- **回合开始时**自动覆盖一份"上回合存档"，防误操作
- **关卡间 + 关卡内剧情节点** 自动存档
- 不存在不可中途退出的长关卡——所有关卡随时可暂停、退出、次日续战
- 存档跨版本兼容自动迁移

## 9. UI/UX 设计规范

### 9.1 关键原则：物理尺寸 > 像素尺寸

**OnGUI 使用物理像素渲染，而手机 DPI（300-500）远高于桌面（96）。**  
一个 30px 的字体在桌面显示器上可读，在手机上只有 1.5mm 高——完全看不清。

**解决方案**：放弃 `_scale` 缩放公式，全部使用**固定大字号**，并引入 DPI 补偿。

### 9.2 字号规范（固定像素值，不乘以 _scale）

| 层级 | 用途 | 字号 | 示例 |
|------|------|------|------|
| **XXL** | 页面大标题 | **48px** | "🏯 征战天下" |
| **XL** | 角色名、重要数字 | **40px** | "李世民" "Lv.3" |
| **L** | 分类标题、按钮 | **32px** | "选择出战武将" "确认阵容" |
| **M** | 正文、属性值 | **28px** | "武82 统95 智88" |
| **S** | 辅助信息 | **22px** | "敌方5人" "出场: ..." |
| **XS** | 角落信息、调试 | **16px** | 版本号 |

> 所有字号为**固定 px 值**，不乘 `_scale`。OnGUI 无法 DPI 感知，固定大字号是唯一可靠的方案。

### 9.3 触控目标规范

| 元素 | 最小尺寸 | 间距 |
|------|---------|------|
| 按钮（底部操作） | **300×60px** | 间隔 ≥20px |
| 关卡卡片（可点击） | 全宽 × **110px** | 间隔 12px |
| 角色卡片（可点击） | 全宽 × **110px** | 间隔 10px |
| 勾选框（✅/☐） | **48×48px** | 点击区域 ≥60px |
| 装备卡（可点击） | 全宽 × **80px** | 间隔 8px |
| 装备选择列表项 | 全宽 × **60px** | — |

> Unity OnGUI 的 GUI.Button 自动处理触控，只需保证 Rect 尺寸 ≥ 上述值。

### 9.6 交互规范
- **点击格子**：触摸→屏幕坐标→世界坐标→六边形坐标→命中检测
- **选中单位**：点击己方单位→黄色高亮环→显示移动范围
- **移动**：点击高亮格→沿A*路径平滑移动（0.15s/格）
- **攻击/计策**：选中单位→底部按钮面板→选择目标
- **列表滚动**：触摸滑动（关卡选择/编组角色）
- **按钮触控**：所有按钮物理尺寸 ≥ 300×60px
- **加载状态**：战斗过渡无加载状态（同一场景）
- **空状态**：无存档时"继续游戏"灰色禁用

### 9.7 代码实现规范

所有 UI 使用**固定像素值**，不乘以 `_scale`：

```csharp
// ❌ 错误：×_scale 会导致在高 DPI 手机上字体过小
GUI.Label(new Rect(0, 0, 200, 30), "text",
    Theme.MakeLabel((int)(24 * _uiScale), ...));

// ✅ 正确：固定大字号，不再依赖 _scale
GUI.Label(new Rect(0, 0, 400, 48), "text",
    Theme.MakeLabel(40, FontStyle.Bold, ...));
```

| 旧方案 | 新方案 |
|--------|--------|
| `fontSize = (int)(24 * _uiScale)` | `fontSize = 40` |
| `cardH = 100 * _uiScale` | `cardH = 110` |
| `(int)(32 * _uiScale)` | `48` |
| `(int)(16 * _uiScale)` | `28` |

## 11. 商业化方案（MVP 不做，v1.1+ 规划）

### 核心策略：广告播放变现
纯单机免费下载，不设付费墙、不卖数值，通过广告播放获取收入。

### 广告类型（按优先级）
| 优先级 | 广告类型 | 形式 | 预期接入版本 |
|---|---|---|---|
| P0 | **激励视频广告** | 玩家主动选择观看，获得资源/复活/增益（每关限3次） | v1.1 |
| P1 | **Banner 广告** | 关卡结算页、主菜单底部，不干扰操作 | v1.2 |
| P2 | **插屏广告** | 关卡开始前 loading 时展示，每次游戏不超过1次 | v1.3 |

### 设计原则
- 所有广告**不可打断战斗进行中**——只在非战斗界面展示
- 激励视频让玩家**主动选择**是否观看，且收益明确告知（"看视频获得双倍金钱？"）
- 不设"去广告"内购——尊重纯单机体验，广告是自愿行为
- 尊重国内用户习惯，不采用欧美式重度广告策略
- Banner/插屏频率严格控制，以不影响游戏体验为底线

## 12. 风险
- 美术风格统一性（免费素材拼接易乱）→ 立项早期定调
- 战斗 AI 智能度 → 预留 30% token 调优
- 数值平衡 → 早期内测组 5 人
